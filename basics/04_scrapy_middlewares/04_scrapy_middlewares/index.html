    解释¶

    在这个中间件我们可以随机更换或添加我们的需求

    当每个request通过下载中间件时，改方法被调用。process_request()必须返回其中之一：返回None、返回个Response对象，返回一个Request对象或raise lgnoreRequest。最常使用的是None

    1. 如果返回None。会将处理过后的reques 丢给中间件链中的下一个中间件爱你的process_request()方法处理，知道丢到下载器，由下载器下载. ==就如上那个更换ip的就是返回None==
    2. 如果返回Response对象，==Scrapy将不会调用任何其他的process_request() 或process_exception()方法==，就不会丢到下载器下载，直接将其返回的resionse丢到中间件链的process_response()处理。可以通过scrapy.http.Response构建Response

    from selenium import webdriver
from scrapy.http import HtmlResponse
import time

class SeleniumMiddleware(object):
    def __init__(self):
        self.driver = webdriver.Chrome()

    def process_request(self, request, spider):
        self.driver.get(request.url)
        time.sleep(2)
        body = self.driver.page_source
        return HtmlResponse(self.driver.current_url,
                           body=body,
                           encoding='utf-8',
                           request=request)

这里我们将请求换成使用seleium去爬取，不会调用其他的process_request()和process_exception()方法
    1. 如果其返回Request 对象，Scrapy则停止调用process_request方法并重新调度返回的reque,当心返回的request被执行后，相应地中间件链将会根据下载的response被调用，相应地中间链将会根据下载的response被调用
    2. 如果其raise一个lgnoreRequest异常，则安装的下载中间件的process_exception()方法会被调用，如果没有任何一个方法处理该异常，则request的errback(Request.errback)方法会被调用。如果没有代码处理抛出的异常，则该异常被忽略且不记录（不同于其他异常那样）

    # 这个用fake随机更换访问头，一般爬虫都会加的，方便反扒
from fake_useragent import UserAgent
class RandomUserAgentMiddlware(object):
    #随机更换user-agent
    def __init__(self, crawler):
        super(RandomUserAgentMiddlware, self).__init__()
        self.ua = UserAgent()
        self.ua_type = crawler.settings.get("RANDOM_UA_TYPE", "random")

    @classmethod
    def from_crawler(cls, crawler):
        return cls(crawler)

    def process_request(self, request, spider):
        def get_ua():
            return getattr(self.ua, self.ua_type)

        request.headers.setdefault('User-Agent', get_ua())

response (Response 对象) – 被处理的response

    解释¶

    当下载的response返回时，process_response()被调用，且 必须返回一下之一，返回一个Response对象、返回一个request对象或raise一个lgnoreRequest 异常 请求参数

    1. 如果将其返回一个response对象(可以与传入的response 相同，也可以是全新的对象)，该response会被在链中的其他中间件process_response()方法处理
    2. 如果返回一个request对象，则中间件链停止，返回的request会被重写调度下载，处理雷诗雨process_request() 返回request所做的那样
    3. 如果其抛出一个lgnorRequest 异常，则调用request的errback(Request.errback)。如果没有代码处理抛出的异常，则该异常被忽略不记录

from scrapy.utils.response import response_status_message


class CustomRetryMiddleware(RetryMiddleware):


    def process_response(self, request, response, spider):

        if request.meta.get('dont_retry', False):
            return response
        if response.status in self.retry_http_codes:
            reason = response_status_message(response.status)
            #如果返回了[500, 502, 503, 504, 522, 524, 408]这些code，换个proxy试试
            proxy = random.choice(proxy_list)
            request.meta['proxy'] = proxy

    from scrapy.downloadermiddlewares.retry import RetryMiddleware
from scrapy.utils.response import response_status_message


class CustomRetryMiddleware(RetryMiddleware):


    #RetryMiddleware类里有个常量，记录了连接超时那些异常
    #EXCEPTIONS_TO_RETRY = (defer.TimeoutError, TimeoutError, DNSLookupError,
    #                       ConnectionRefusedError, ConnectionDone, ConnectError,
    #                       ConnectionLost, TCPTimedOutError, ResponseFailed,
    #                       IOError, TunnelError)
    def process_exception(self, request, exception, spider):
        if isinstance(exception, self.EXCEPTIONS_TO_RETRY) and not request.meta.get('dont_retry', False):
            #这里可以写出现异常那些你的处理            
            proxy = random.choice(proxy_list)
            request.meta['proxy'] = proxy
            time.sleep(random.randint(3, 5))
            return self._retry(request, exception, spider)
    #_retry是RetryMiddleware中的一个私有方法，主要作用是
    #1.对request.meta中的retry_time进行+1 
    #2.将retry_times和max_retry_time进行比较，如果前者小于等于后者，利用copy方法在原来的request上复制一个新request，并更新其retry_times，并将dont_filter设为True来防止因url重复而被过滤。
    #3.记录重试reason
    1. 如果其返回 None ，Scrapy将会继续处理该异常，接着调用已安装的其他中间件的 process_exception() 方法，直到所有中间件都被调用完毕，则调用默认的异常处理。
    2. 如果其返回一个 Response 对象，则已安装的中间件链的 process_response() 方法被调用。Scrapy将不会调用任何其他中间件的 process_exception() 方法。
    3. 如果其返回一个 Request 对象， 则返回的request将会被重新调用下载。这将停止中间件的 process_exception() 方法执行，就如返回一个response的那样。

    编写自定义spider中间件¶

    • process_spider_input(response, spider)

    这个方法时在下载器中间件处理完成后，马上要进入某个回调函数parse_xxx()前调用。process_spider_output(response, result, output)在这个方法处理完成以后，数据如果是item，就会被交给pipeline；如果是请求，就会被交给调度器，然后下载器中间件才会开始运行。这个方法的参数result就是爬虫爬出来的item或者scrapy.Request()。由于yield得到的是一个生成器，生成器是可以迭代的，所以result也是可以迭代的，可以使用for循环来把它展开。

    • process_spider_output(response, result, spider)

    是在爬虫运行yield item 或是yield scrapy.Request() 的时候被调用。一般返回result

    • process_spider_exception(response, exception, spider)

    当spider中间件抛出异常时，这个方法被调用，返回None或可迭代对象的Request,dict,item
